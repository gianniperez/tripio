# Preparar PR

Prepara los cambios como Pull Request listo para review.

1. **Checklist pre-PR**: ¿Pasan todos los tests? ¿El código cumple con el linter? ¿Hay resumen de cambios?

2. **Formato de descripción de PR**: Incluir Resumen, Cambios (lista), Cómo probar, Puntos ciegos.

3. **Validaciones antes de merge**: Sugerir validaciones que el revisor debería hacer (tests relevantes, impacto en otras partes, documentación actualizada).

4. **Puntos ciegos**: Incluir sección "¿Qué podría fallar en review? ¿Qué no está documentado?"
