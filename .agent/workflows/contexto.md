# Mapa del Sistema

Lee el repo y produce un mapa del sistema. Estructura la respuesta con jerarquía clara.

1. **Arquitectura**: Cómo está organizado el proyecto

2. **Carpetas clave**: Qué contiene cada una

3. **Dependencias**: Librerías, frameworks, servicios externos

4. **Puntos de entrada**: Cómo se ejecuta/inicia el proyecto

5. **Stack técnico**: Lenguajes, frameworks, herramientas

## Formato

Usa estructura jerárquica: Nivel 1 (Visión general), Nivel 2 (Componentes principales), Nivel 3 (Detalles relevantes).

## Sección adicional

Incluye **Posibles mejoras de estructura** si detectas: desorden en carpetas, código duplicado, dependencias circulares, documentación faltante.
